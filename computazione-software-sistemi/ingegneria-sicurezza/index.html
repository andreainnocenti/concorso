    Ingegneria e Sicurezza del Software¶

    📖 Introduzione¶

    L'ingegneria del software fornisce metodologie per sviluppare software di qualità in modo sistematico.

    🔄 Ciclo di Vita del Software (SDLC)¶

    1. Requisiti: raccolta e analisi
    2. Design: architettura e progettazione
    3. Implementazione: sviluppo
    4. Testing: verifica e validazione
    5. Deployment: rilascio
    6. Manutenzione: supporto e aggiornamenti

    🏃 Metodologie Agile¶

    Scrum¶

    • Sprint (2-4 settimane)
    • Daily standup
    • Sprint review e retrospective

    Kanban¶

    • Visualizzazione del flusso
    • Limitazione WIP (Work In Progress)
    • Miglioramento continuo

    🔧 DevOps¶

    Principi: - Continuous Integration (CI) - Continuous Delivery/Deployment (CD) - Infrastructure as Code (IaC) - Monitoring e Logging

    🧪 Testing¶

    Tipi di Test¶

    • Unit Testing: singole unità
    • Integration Testing: interazioni
    • System Testing: sistema completo
    • Acceptance Testing: requisiti utente

    Test-Driven Development (TDD)¶

    Red → Green → Refactor

    🔒 Sicurezza del Software¶

    OWASP Top 10¶

    1. Injection
    2. Broken Authentication
    3. Sensitive Data Exposure
    4. XML External Entities (XXE)
    5. Broken Access Control
    6. Security Misconfiguration
    7. Cross-Site Scripting (XSS)
    8. Insecure Deserialization
    9. Using Components with Known Vulnerabilities
    10. Insufficient Logging & Monitoring

    Secure Coding Practices¶

    • Input validation
    • Output encoding
    • Principle of least privilege
    • Defense in depth

    📚 Risorse¶

    Standard e Framework

    • ISO/IEC 27001 (Security)
    • NIST Cybersecurity Framework
    • OWASP SAMM (Software Assurance Maturity Model)
