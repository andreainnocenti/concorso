# Caratteristiche Funzionali e Non Funzionali del Software

## 📖 Introduzione

Le caratteristiche del software si dividono in funzionali (cosa fa) e non funzionali (come lo fa).

## 🎯 Requisiti Funzionali

Descrivono le funzionalità specifiche che il sistema deve fornire.

**Esempi:**
- Autenticazione utente
- Elaborazione pagamenti
- Generazione report

## ⚙️ Requisiti Non Funzionali

### Performance
- Tempo di risposta
- Throughput
- Latenza

### Scalabilità
- Verticale (più risorse)
- Orizzontale (più istanze)

### Affidabilità
- Disponibilità (uptime)
- Fault tolerance
- Recovery

### Sicurezza
- Confidenzialità
- Integrità
- Autenticazione
- Autorizzazione

### Usabilità
- Facilità d'uso
- Accessibilità
- User experience

### Manutenibilità
- Leggibilità del codice
- Modularità
- Documentazione

## 📊 Metriche di Qualità

- **Complessità Ciclomatica**
- **Code Coverage**
- **Technical Debt**
- **MTBF/MTTR** (Mean Time Between/To Failure/Repair)
